## 📝 وصف المشروع
مشروع ويب تفاعلي مصمم وفق معايير تطبيقات الصفحة الواحدة (Single Page Application)، ويتميز بواجهة مستخدم ديناميكية ومتجاوبة بالكامل (Responsive Design) لتلائم مختلف أبعاد الشاشات (الهواتف، الأجهزة اللوحية، والحواسيب).

النظام تم تقسيمه وهندسته إلى ثلاثة أجزاء أساسية:

1️⃣ واجهة العرض (UI)
شريط ملاحة علوي (Navbar): مخصص للتنقل السلس والذكي داخل أقسام الصفحة.

قسم تعريفي: يضم البيانات الرسمية والتخصص البرمجي لمطور النظام.

تذييل مخصص (Footer): يضمن حقوق التطوير والنشر لعام 2026.

2️⃣ منطق العمل (Business Logic)
نموذج حجز طبي تفاعلي: مبني بالكامل باستخدام كود JavaScript (Vanilla JS).

التحقق من البيانات (Validation): يتم فحص مدخلات المستخدم فورياً ومحلياً (Client-side).

تفاعل ديناميكي: عرض رسائل تأكيد الحجز للمريض فورياً دون الحاجة لإعادة تحميل الصفحة.

3️⃣ إدارة النسخ والمستودعات (Git & GitHub Lifecycle)
تتبع الكود: توثيق مراحل البناء عبر سجل Commits منظم واحترافي التسمية.

إدارة الفروع (Branching): محاكاة بيئة العمل الحقيقية عبر عزل الميزات في فرع feature/design-update.

الدمج والأتمتة: رفع طلب دمج (Pull Request)، إغلاق الملاحظات (Issues) تلقائياً، والنشر المباشر أونلاين عبر GitHub Pages.

## 🛠️ التقنيات المستخدمة
* **HTML5:** لبناء هيكل الصفحة والأقسام المطلوبة.
* **CSS3:** لتنسيق المظهر وإضافة التصميم المتجاوب (Responsive Design).
* **JavaScript (Vanilla JS):** لإضافة المنطق وفحص مدخلات الحجز وعرض رسالة النجاح التفاعلية.
